Why teams reach for it

GitHub Actions is infrastructure. Treat its inputs and outputs like API surface.

Discover contracts automatically

Scan a repository, map workflows to declarations, and surface stable relative paths for each contract file. 

github-actionspec discover --repo /path/to/repo

Validate one contract directly

Point the CLI at schema files, a contract declaration, and an actual payload when you need a focused validation pass. 

github-actionspec validate \
  --schema schema/workflow_run.cue \
  --schema schema/declaration.cue \
  --contract contract.cue \
  --actual actual.json

Validate a workflow from the repo root

Skip manual lookup. Resolve the matching declaration for a workflow name and vet it against the incoming JSON payload. 

github-actionspec validate-repo \
  --repo /path/to/repo \
  --workflow build-infrastructure.yml \
  --actual actual.json

Simple flow

Three steps from declaration to enforcement.

01

Declare the contract

Store CUE files in .github/actionspec/** and pin each declaration to a workflow filename.

02

Collect the actual payload

Feed the validator the JSON your workflow produced or the event payload your system consumed.

03

Run it where your team works

Use the Rust CLI locally, or run the runtime image in CI when you want a consistent environment with cue included.
